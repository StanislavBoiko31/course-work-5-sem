ОСОБЛИВОСТІ АРХІТЕКТУРИ:

1. МОНОЛІТНА АРХІТЕКТУРА
   • Всі модулі працюють в одному процесі Django
   • Не окремі сервіси, а Django apps (модулі)
   • Швидка комунікація (немає мережевих затримок)

2. КОМУНІКАЦІЯ МІЖ МОДУЛЯМИ
   • Прямі імпорти (не HTTP)
   • Приклад: from photographers.models import Photographer
   • Переваги: швидше, простіше, транзакції БД працюють коректно

3. URL РОУТИНГ (як API Gateway)
   • Централізований роутинг в studio/urls.py
   • Розподіляє запити до відповідних модулів
   • Обробка статичних файлів та медіа

4. БАЗА ДАНИХ
   • PostgreSQL (може бути локально або віддалено)
   • Одна база даних для всієї системи
   • Всі модулі зберігають та отримують дані з БД
